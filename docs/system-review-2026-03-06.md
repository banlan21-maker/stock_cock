# 주식콕 시스템 전체 리뷰 (2026-03-06)

## 목표 흐름

사용자가 의도한 투자 가이드 흐름:

```
테마트렌드 → 키워드피드 → 공시확인 → 종목콕(비교/AI분석) → 포트폴리오(평가/일지)
```

실제 앱 내비게이션 구조:

```
대시보드(테마+키워드피드) | 이슈콕 | 정책콕 | 공시콕 | 종목콕 | 포트폴리오
```

## 구조적 문제 목록

### [P1] 테마 클릭 → 다음 단계 연결 없음 (최우선)

**문제**: 대시보드 테마트렌드에서 관련 종목을 보여주지만, 종목 항목을 클릭해서 상세 페이지로 이동하는 링크가 없거나 불명확함.

초보 투자자 관점에서 "이 종목을 어떻게 더 알아보지?"에서 흐름이 단절됨.

**확인 사항**: 테마트렌드 카드 내 종목 항목에 `/stock/[code]` 링크 존재 여부

**해결 방향**:
- 테마 종목 항목 클릭 시 `/stock/[code]` 이동
- "이 테마 관련 뉴스 보기" 버튼 → 이슈콕으로 연결

**관련 파일**:
- `frontend/components/dashboard/ThemeTrend.tsx`
- `backend/app/routers/dashboard.py`

---

### [P2] 키워드 시스템 - 신규 사용자 닭/달걀 문제

**문제**:
- 키워드는 `/stock/[code]` 상세 페이지에서 추가 가능
- 이슈콕/정책콕은 그 키워드로 필터링
- 처음 온 사용자 → 종목 상세 방문 전 → 키워드 없음 → 전체 뉴스 나열 → 방향 상실

**해결 방향**:
- 대시보드 키워드피드 빈 상태일 때: "아직 키워드가 없어요. 테마 종목을 눌러 키워드를 추가해보세요" 안내 추가
- 이슈콕/정책콕 첫 진입 시에도 동일 안내

**관련 파일**:
- `frontend/components/dashboard/KeywordFeed.tsx`
- `frontend/app/(app)/issues/page.tsx`
- `frontend/app/(app)/policy/page.tsx`
- `frontend/lib/customKeywords.ts`

---

### [P3] 공시콕 페이지의 역할 모호성

**문제**:
- `/disclosure` (공시콕): 오늘 전체 기업 공시 목록 → "탐색" 용도
- `/stock/[code]` 공시 탭: 해당 종목 공시 → "확인" 용도

사용자가 의도한 흐름의 "공시확인"은 특정 종목의 공시 확인을 의미함.
공시콕에서 공시 항목 클릭 시 해당 종목 상세 페이지로 이동하는 링크가 있어야 함.

**확인 사항**: 공시콕 항목 클릭 → `/stock/[code]` 이동 여부

**관련 파일**:
- `frontend/app/(app)/disclosure/page.tsx`
- `frontend/components/disclosure/DisclosureCard.tsx` (있다면)

---

### [P4] AI 포트폴리오 진단 진입 경로 숨겨짐

**문제**: 보유종목이 있을 때만 "AI 진단" 버튼이 표시됨. 첫 종목 추가 후 자동 안내 없음.

```tsx
// frontend/app/(app)/portfolio/page.tsx:222
{holdings.length > 0 && (
  <Link href="/portfolio/analysis">AI 진단</Link>
)}
```

**해결 방향**: 첫 종목 추가 완료 시 "AI 진단을 해보세요!" CTA 표시 또는 빈 상태 안내에 설명 추가

---

### [P5] 이슈콕 뉴스 → 종목 연결 약함

**문제**: 이슈콕에서 뉴스를 읽은 후 관련 종목으로 직접 이동하는 링크가 명확하지 않으면 뉴스 → 종목 탐색 흐름이 단절됨.

**확인 사항**: NewsCard 컴포넌트에 관련 종목 링크 존재 여부

**관련 파일**:
- `frontend/components/issues/NewsCard.tsx` (또는 유사 파일)
- `backend/app/routers/issues.py`

---

### [P6] 대시보드 → 이슈콕/정책콕 자연스러운 연결 없음

**문제**: 대시보드에서 테마를 확인한 후 관련 뉴스/정책을 보려면 수동으로 이슈콕/정책콕 탭을 클릭해야 함. 자연스러운 흐름 안내 없음.

**해결 방향**: 테마 카드 하단에 "관련 뉴스 보기 →" 또는 "관련 정책 보기 →" 링크 추가

## 성능 현황

| 항목 | 상태 | 비고 |
|------|------|------|
| 테마트렌드 pykrx 300회 호출 | 수정됨 | dict lookup으로 교체 |
| 상장폐지 종목 fail cache (10분 TTL) | 수정됨 | - |
| fdr.StockListing 서킷브레이커 (30분) | 수정됨 | - |
| yfinance `.info` 제거 | 수정됨 | - |
| KIS API (클라우드에서 차단) | 구조적 한계 | Firebase Cloud Run은 한국 IP 아님 → 사실상 전 종목 yfinance 사용 중 |
| 주식목록 캐시 pre-warm | 미구현 | 첫 요청 시 여전히 느림 |

### KIS API 관련 근본 한계

Firebase Cloud Run 환경은 한국 IP가 아니어서 KIS API 접근 자체가 차단됨.
현재 폴백 체인: KIS → FDR → yfinance 에서 사실상 yfinance만 동작.

**옵션**:
1. KIS 제거하고 pykrx + yfinance 전용으로 단순화
2. 한국 내 서버(AWS Seoul, Naver Cloud 등)로 백엔드 이전
3. 별도 주가 데이터 API 서비스 사용 (예: 한국투자증권 OpenAPI 외 대안)
